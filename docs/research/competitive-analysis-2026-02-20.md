**评估人**: Research Ben Thompson

## 执行摘要

Ent-DNS 处于企业 DNS 过滤市场，主要竞品为 AdGuard Home（家用/SOHO）、Pi-hole（开源社区）、Blocky（高性能）。Ent-DNS 的核心差异化优势在于：Rust 高性能架构、现代 Web UI、完整的 RBAC + 审计日志。

**市场定位**：
- **目标市场**：中小企业（10-1,000 设备）、家庭高级用户
- **竞争优势**：性能优于 Go 实现（Pi-hole/Blocky）、安全优于 AdGuard Home（无 RBAC）
- **市场空白**：企业级 RBAC + 审计日志 + 现代化 Web UI

**风险因素**：
- DoH/DoT 缺失（现代浏览器默认使用 DoH）
- AdGuard Home 品牌认知度高
- Cloudflare for Teams 企业级产品竞争

## 二、竞品深度分析

### 2.1 AdGuard Home

**市场定位**：家用/SOHO 级 DNS 过滤服务器

**优势**：
- ✅ 功能全面（DoH/DoT/DNSCrypt、时间段规则、家长控制）
- ✅ 部署简单（Docker、一键安装脚本）
- ✅ 品牌认知度高（AdGuard 生态）
- ✅ 移动端 App 支持

**劣势**：
- ❌ 无 RBAC（单 admin 用户）
- ❌ 无审计日志（不满足企业合规）
- ❌ 无多租户支持
- ❌ 性能中等（Go 实现）
- ❌ 闭源核心（商业公司）

**市场份额**：
- GitHub Stars: ~20K（截至 2026-02）
- Docker Hub 下载量: ~10M
- 主要市场：家庭用户、小型工作室

**竞争策略**：
- **避开**：功能全面度（AdGuard Home 功能太多）
- **聚焦**：企业级管控（RBAC + 审计日志）
- **超越**：性能（Rust vs Go）

### 2.2 Pi-hole

**市场定位**：开源网络广告拦截器

**优势**：
- ✅ 开源社区活跃（GitHub Stars ~50K）
- ✅ 部署简单（Raspberry Pi、Docker）
- ✅ 社区插件丰富（Teleporter、AdminLTE）
- ✅ 文档完善

**劣势**：
- ❌ 性能低（PHP + FTL）
- ❌ 无 RBAC
- ❌ 无审计日志
- ❌ 功能老旧（无 DoH/DoT）
- ❌ UI 设计过时

**市场份额**：
- GitHub Stars: ~50K（截至 2026-02）
- Docker Hub 下载量: ~100M
- 主要市场：家庭用户、极客社区

**竞争策略**：
- **超越**：性能（Rust vs PHP）、UI 现代化
- **对标**：功能完整度（DoH/DoT 是准入门槛）

### 2.3 Blocky

**市场定位**：高性能 DNS 过滤服务器（Go 实现）

**优势**：
- ✅ 性能高（Go + 并发优化）
- ✅ 支持正则表达式
- ✅ 支持条件规则
- ✅ 配置灵活（YAML）

**劣势**：
- ❌ 无 Web UI（仅配置文件管理）
- ❌ 无 RBAC
- ❌ 无审计日志
- ❌ 社区规模小（GitHub Stars ~3K）

**市场份额**：
- GitHub Stars: ~3K（截至 2026-02）
- 主要市场：极客、开发者

**竞争策略**：
- **超越**：Web UI 用户体验
- **对标**：功能完整度（正则、条件规则）

### 2.4 Cloudflare for Teams

**市场定位**：企业级 SaaS DNS 安全服务

**优势**：
- ✅ 企业级功能（SSO、多租户、SIEM 集成）
- ✅ 全球 CDN（低延迟）
- ✅ 品牌信任度高（Cloudflare）
- ✅ 免费层可用

**劣势**：
- ❌ 闭源 SaaS（无数据主权）
- ❌ 自定义能力有限
- ❌ 定价昂贵（企业版 $5/用户/月）

**竞争策略**：
- **差异化**：自托管（数据主权）、可定制
- **避开**：SaaS 便捷性

## 四、差异化战略

### 4.1 核心差异化（3 个维度）

**维度 1：性能优势（技术差异化）**
- Rust 异步架构，QPS 10,000+（Go 实现 5,000+，PHP 1,000+）
- 内存占用低（~50MB vs Pi-hole ~200MB）
- 适合高并发场景（企业内网）

**维度 2：企业管控（功能差异化）**
- RBAC 权限管理（3 角色）
- 审计日志（操作追溯）
- Prometheus 指标（运维可观测）

**维度 3：用户体验（UX 差异化）**
- 现代 Web UI（React + Tailwind + shadcn/ui）
- 实时查询日志（WebSocket）
- 批量操作（规则管理效率高）

### 4.2 差异化定位地图

```
          高企业功能
                ↑
                |    Cloudflare for Teams
                |        (SaaS)
                |
    AdGuard Home ---+--- Ent-DNS
    (家用)          |
                |
                |    Pi-hole / Blocky
                |        (开源)
                |
                ↓
          低企业功能
```

**解释**：
- Cloudflare for Teams：企业功能最高，但闭源 SaaS
- AdGuard Home：家用/SOHO，无企业功能
- Pi-hole/Blocky：开源，但无企业功能
- Ent-DNS：平衡点（企业功能 + 开源自托管）

### 4.3 竞争防御策略

| 竞品 | 防御策略 |
|------|---------|
| AdGuard Home | 聚焦企业级管控（RBAC + 审计日志），避开家用场景 |
| Pi-hole | 强调性能（Rust vs PHP）、UI 现代化 |
| Cloudflare for Teams | 强调数据主权、自托管、可定制 |

### 5.2 市场空白识别

**空白 1：企业级自托管 DNS 过滤器**
- 现状：AdGuard Home 无 RBAC，Cloudflare for Teams 闭源
- 机会：Ent-DNS 提供 RBAC + 审计日志 + 开源自托管

**空白 2：高性能企业 DNS 过滤器**
- 现状：Pi-hole 性能低（PHP），Blocky 无 Web UI
- 机会：Ent-DNS 提供 Rust 高性能 + 现代 Web UI

**空白 3：可定制企业 DNS 过滤器**
- 现状：SaaS 产品自定义能力有限
- 机会：Ent-DNS 提供 API 开放、开源可定制

**阶段 2：中小企业早期用户（6-12 个月）**
- 提供免费层（10 设备）
- 收集用户反馈，快速迭代
- 建立用户案例

**阶段 3：商业化（12-24 个月）**

## 七、结论与建议

### 7.1 市场定位总结

Ent-DNS 定位于**企业级自托管 DNS 过滤器**，核心差异化在于：
- **性能优势**：Rust 异步架构，QPS 10,000+
- **企业管控**：RBAC + 审计日志（竞品缺失）
- **用户体验**：现代 Web UI + 实时日志

目标市场：中小企业（10-1,000 设备）、教育机构、家庭高级用户。

1. **DoH/DoT 支持**（2-3 周）—— 准入门槛
