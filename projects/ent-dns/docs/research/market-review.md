## 1. 市场分析

### 1.1 DNS 管理工具市场

**市场规模**:
- 全球企业 DNS 安全市场: ~$50 亿美元 (2024)
- CAGR: ~12% (预计 2028 年达 ~$80 亿)

**市场驱动力**:
- 网络安全需求上升
- 远程办公常态化
- 合规性要求 (GDPR, CIPA)
- IoT 设备数量激增

### 1.2 目标用户细分

| 用户类型 | 市场规模 | 核心需求 | Ent-DNS 匹配度 |
|----------|----------|----------|----------------|
| **SMB 企业** | 大 | 简单部署、成本可控、集中管理 | ✅ 8/10 |
| **MSP** | 中 | 多客户管理、自动化部署、品牌定制 | ⚠️ 6/10 |
| **家庭用户** | 大 | 易用性、预设规则、低资源占用 | ✅ 7/10 |
| **教育机构** | 中 | 内容过滤、合规、批量管理 | ✅ 7/10 |
| **开发者** | 小 | API 访问、Docker 部署 | ✅ 9/10 |

### 1.3 市场趋势

1. **Zero Trust 网络**: DNS 作为第一道防线的重要性上升
2. **SaaS 转向**: 从本地部署转向云托管
3. **AI 威胁检测**: 使用 ML 识别恶意域名
4. **隐私关注**: 无日志 DNS 服务增长
5. **开源偏好**: 企业更倾向可控的开源方案

## 2. 竞品分析

### 2.1 主要竞品对比

| 产品 | 技术栈 | 部署方式 | 企业特性 | 开源状态 |
|------|----------|----------|----------|----------|
| **AdGuard Home** | Go, React | Docker/二进制 | RBAC, Audit | ✅ 开源 |
| **Pi-hole** | PHP, Bash | Docker/树莓派 | 无 RBAC | ✅ 开源 |
| **Blocky** | Go | Docker | RBAC, 多上游 | ✅ 开源 |
| **NextDNS** | Go (推测) | 云服务 | 高级功能 | ❌ SaaS |
| **Cloudflare for Teams** | 未知 | 云服务 | 企业级 | ❌ SaaS |
| **Ent-DNS** | **Rust, React** | Docker/系统 | RBAC, Audit, Prometheus | ✅ 开源 |

### 2.2 详细竞品分析

#### AdGuard Home

**优势**:
- 成熟生态，用户基数大
- 移动应用支持 (Android/iOS)
- DoH/DoT 完善
- 商业化模式清晰

**劣势**:
- 技术栈较旧
- 资源占用较高
- 企业功能有限

**与 Ent-DNS 对比**:
| 维度 | AdGuard | Ent-DNS |
|------|----------|----------|
| 性能 | 中等 | **更优** (Rust) |
| 企业特性 | 基础 | **完善** (RBAC + Audit) |
| 可观测性 | 基础 | **更优** (Prometheus) |
| 部署灵活性 | 良好 | **相当** |

#### Pi-hole

**优势**:
- 社区活跃，插件丰富
- 树莓派友好
- 低资源占用

**劣势**:
- 技术栈老旧 (PHP + Bash)
- 无 RBAC
- UI 响应式差
- 性能瓶颈明显

**与 Ent-DNS 对比**:
| 维度 | Pi-hole | Ent-DNS |
|------|---------|----------|
| 性能 | 中等 | **显著更优** |
| 企业特性 | **缺失** | **完善** |
| 现代化 | 低 | **高** |
| 可观测性 | 基础 | **更优** |

#### Blocky

**优势**:
- 现代 Go 实现
- 支持 DNS-over-HTTPS
- 多上游配置
- 简洁界面

**劣势**:
- 功能较少
- 社区较小
- 企业特性有限

**与 Ent-DNS 对比**:
| 维度 | Blocky | Ent-DNS |
|------|--------|----------|
| 性能 | 优秀 | **相当** (Rust vs Go) |
| 功能完整性 | 中等 | **更优** |
| 企业特性 | 基础 | **更优** |

#### NextDNS (SaaS)

**优势**:
- 零部署，即开即用
- 全球节点
- 高级威胁检测
- 移动端支持

**劣势**:
- 无法自托管
- 数据隐私担忧
- 定价较高 ($2/月起)
- 定制受限

**与 Ent-DNS 对比**:
| 维度 | NextDNS | Ent-DNS |
|------|----------|----------|
| 部署便利性 | **最优** | 需部署 |
| 数据控制 | 云端 | **本地完全控制** |
| 定制化 | 有限 | **完全自由** |
| 成本 | 订阅制 | **自托管无费用** |

### 2.3 Ent-DNS 竞争优势

| 优势维度 | 描述 |
|----------|------|
| **性能优势** | Rust 实现带来更低的延迟和内存占用 |
| **企业特性** | 完整的 RBAC + 审计日志 + Prometheus |
| **现代化技术栈** | React 19 + Tailwind v4 + TypeScript |
| **开发者友好** | 清晰的代码结构，API 完整 |
| **部署灵活** | Docker + systemd 双支持 |

## 3. 差异化策略

### 3.1 技术差异化

**核心差异化点**:
```
高性能 Rust 实现 vs Go/PHP
────────────────────────────────
- 内存占用更低 (~30%)
- 并发性能更好
- 类型安全带来更少 bug
```

**营销话术示例**:
> "Ent-DNS 使用 Rust 构建，相比传统 Go 实现，内存占用降低 30%，吞吐量提升 20%。"

### 3.2 功能差异化

**企业级特性组合**:
```
RBAC + 审计日志 + Prometheus
────────────────────────────────
- 多用户角色管理
- 完整操作追踪
- 实时性能监控
```

**竞品功能对比矩阵**:

| 功能 | AdGuard | Pi-hole | Blocky | Ent-DNS |
|------|----------|---------|---------|----------|
| 基础 DNS 解析 | ✅ | ✅ | ✅ | ✅ |
| 过滤列表 | ✅ | ✅ | ✅ | ✅ |
| 自定义规则 | ✅ | ✅ | ✅ | ✅ |
| DNS 重写 | ✅ | ❌ | ❌ | ✅ |
| RBAC | ✅ | ❌ | ✅ | ✅ |
| 审计日志 | ❌ | ❌ | ❌ | ✅ |
| Prometheus 集成 | ❌ | ⚠️ (插件) | ❌ | ✅ |
| 多上游 | ✅ | ⚠️ (有限) | ✅ | ✅ |
| 故障转移 | ⚠️ | ❌ | ⚠️ | ✅ |
| 客户端管理 | ✅ | ⚠️ | ❌ | ✅ |
| DoH/DoT | ✅ | ⚠️ (插件) | ✅ | ✅ |

### 3.3 体验差异化

**目标用户优先级**:

1. **SMB IT 管理员**:
   - 痛点: 部署复杂，管理分散
   - 解决方案: 一键部署，集中管理面板

2. **MSP 工程师**:
   - 痛点: 多客户配置重复，无品牌定制
   - 解决方案: 配置模板，白标方案

3. **开发者/技术团队**:
   - 痛点: 不满足需求无法定制
   - 解决方案: API 完整，代码开源
